# Nomad 单节点安装脚本交付说明

## 变更范围

- 重写 Nomad 安装脚本，入口为 `tools/nomad/manager.sh`。
- 支持 `install`、`uninstall`、`help` 子命令。
- 支持通过 `install --version VERSION` 或 `install VERSION` 指定 Nomad 版本。
- 未指定版本时，从 HashiCorp releases 页面解析最新 OSS 版本；解析失败时回退到默认版本 `2.0.0`。

## 安装行为

- 下载 `nomad_${version}_linux_${arch}.zip`。
- 下载并校验 `nomad_${version}_SHA256SUMS`。
- 优先用 `unzip` 解压；目标机没有 `unzip` 时，回退使用 `python3 -m zipfile` 解压。
- 安装二进制到 `/usr/local/bin/nomad`。
- 创建 `nomad` 系统用户。
- 创建 `/etc/nomad.d`、`/opt/nomad`、`/opt/nomad/data`。
- 写入 systemd 服务 `/etc/systemd/system/nomad.service`。
- 写入单节点配置 `/etc/nomad.d/nomad.hcl`，同时启用 server 和 client，`bootstrap_expect = 1`。
- 默认写入托管配置片段 `/etc/nomad.d/40-telemetry.hcl` 和 `/etc/nomad.d/80-docker.hcl`。
- 启用 ACL，并在启动后尝试执行 `nomad acl bootstrap`，Token 保存到当前执行用户的 `~/nomad.acl`。

## 卸载行为

- 停止并禁用 `nomad.service`。
- 删除 `/etc/systemd/system/nomad.service`、`/usr/local/bin/nomad`、`/etc/nomad.d`、`/opt/nomad`。
- 删除带有 `# Generated by installNomad.sh` 标记的 `~/nomad.acl`，避免卸载后残留敏感 Token。
- 执行 `systemctl daemon-reload` 和 `systemctl reset-failed nomad`，避免卸载后残留 failed 状态。
- 删除 `nomad` 系统用户。
- 删除前对关键路径做保护，拒绝删除根目录、`/usr`、`/etc`、`/opt` 等危险路径。

## 配置托管约束

- 所有配置子命令只管理首行为 `# Managed by tools/nomad/manager.sh` 的文件。
- 写入或删除配置后，先执行 `nomad config validate /etc/nomad.d`。
- 校验通过后重启 `nomad.service`。
- 校验或重启失败时恢复原配置文件。
- `plugin "docker"` 和 `plugin "raw_exec"` 使用完整托管片段，避免同名 plugin block 覆盖导致行为不明确。

## Vault JWT Profile

- `vault-jwt apply` 会保存 profile 到 `/var/lib/installNomad/vault-jwt/<PROFILE>.env`。
- profile 记录 `VAULT_ADDR`、`NOMAD_ADDR_PROFILE`、`NOMAD_JWKS_URL`、`VAULT_AUTH_PATH`、`VAULT_ROLE`、`VAULT_POLICY`、`AUDIENCE`、`TTL`、`SECRET_PATHS`。
- `vault-jwt apply` 会先写 Nomad `vault` 配置，等待 Nomad JWKS URL 可访问，再启用或复用 Vault JWT auth path，然后写 Vault JWT config、policy、role。
- `vault-jwt apply` 写入 role 时使用 JSON payload，包含 `claim_mappings`、`token_type=service` 和 `token_explicit_max_ttl=0`。
- 已存在 profile 且传入参数不同会拒绝覆盖，除非加 `--force`。
- 测试或特殊部署可通过环境变量 `VAULT_JWT_PROFILE_DIR` 覆盖 profile 目录。
